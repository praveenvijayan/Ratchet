# Architecture map

_Machine-generated by /ratchet-init on 2026-07-08. Provisional — the code wins._

## Project type

This repository IS the Ratchet framework — a GitHub-native delivery loop for
coding agents. It is not an application: no package manifest, no build step.
It ships as markdown contracts, zero-dependency Node (ESM, Node 20+) scripts,
shell scripts, and GitHub Actions workflows, distributed as a template repo
and an optional Claude Code plugin.

## Source layout

- Root — the behavioural contracts and docs: `AGENTS.md` (canonical operating
  manual), `CLAUDE.md` / `GEMINI.md` (thin pointers to it), `GATES.md`
  (project-owned verification gates), `DOCS.md` (full documentation),
  `README.md`, `setup.sh` (skill-mirror sync), `.ratchet-version`,
  `.env.example`.
- `plan/` — the plan-file format contract (`README.md`) plus one worked
  example. Plan files compile to GitHub issues.
- `memory/` — the three durable-memory files (`USER.md`, `MEMORY.md`, this map).
- `scripts/` — the executable core: `plan-sync.mjs` (plan→issue compiler) with
  `plan-sync.test.mjs` (its regression test), `run-gates.mjs` (the GATES.md
  gate runner shared by local verify and CI) with `run-gates.test.mjs`,
  `criteria.mjs` (shared acceptance-criteria readiness rule) and
  `sweep-lease.mjs` (shared lease-freshness/heartbeat rule) with their tests,
  `ratchet-watch.sh`/`.mjs` (local webhook bridge for the continuous loop),
  `ratchet-update.sh` and `ratchet-uninstall.sh` (framework lifecycle).
- `.github/workflows/` — the automation edge of the loop: `plan-sync`,
  `unblock-dependents`, `sweep-stale-claims`, `pr-gates` (runs the GATES.md
  gates server-side on every agent PR), and the opt-in `ratchet-run`.
- `.agents/skills/` — the CANONICAL skill sources (one dir per `ratchet-*`
  skill: `SKILL.md` body + `agents/openai.yaml` Codex policy).
- `.claude/skills/` and `plugin/skills/` — generated mirrors of the canonical
  skills for Claude Code and the plugin package. Never edited directly.
- `.claude-plugin/` — plugin marketplace manifest (Claude Code only).

## Major components

- **Operating manual** (`AGENTS.md`) — the 7-step loop every agent follows:
  pick → claim → build → verify → PR → rework → system-close. 100% framework,
  overwrite-safe on update.
- **Plan compiler** (`scripts/plan-sync.mjs`) — deterministic, idempotent
  `plan/*.md` → issues sync; dedupes via a `plan-id` marker in issue bodies.
- **State-machine automation** (`.github/workflows/`) — reacts to merges and
  closes: unblocks dependents, sweeps stale claims, optionally runs an agent
  in CI.
- **Local loop bridge** (`scripts/ratchet-watch.*`) — forwards PR/review
  events to the local agent via `gh webhook forward`.
- **Gate runner** (`scripts/run-gates.mjs`) — parses the `GATES.md` table and
  runs the gates in order, fail-fast, skipping `TODO:` rows. Single source of
  truth invoked by both local verify and the `pr-gates` CI check.
- **Skills** (nine `ratchet-*` dirs) — slash-command ergonomics: init, plan,
  sync, next, status, memory, map, update, uninstall.

## Conventions

- Canonical-source + mirror: edit `.agents/skills/`, then run `./setup.sh` to
  regenerate `.claude/skills/` and `plugin/skills/`. Mirrors are never edited.
- Zero dependencies everywhere: scripts are plain Node 20+ ESM or POSIX shell;
  workflows inline their logic via `actions/github-script`.
- Markdown-as-config: behaviour lives in prose contracts (`AGENTS.md`,
  `plan/README.md`, `GATES.md`), not code, so any agent tool can read it.
- Framework vs project-owned split: updates overwrite framework paths only;
  `GATES.md`, `memory/`, `plan/*.md`, `.env`, `README.md` are never touched.
- Version bumps update `.ratchet-version` and the README badge together.

## Not yet present

- No lint/format tooling for the scripts or markdown.
- The decision logic of both `unblock-dependents` and `sweep-stale-claims` is
  now covered via shared helpers (`scripts/criteria.mjs`, `scripts/sweep-lease.mjs`)
  with their tests; the workflows' GitHub-API glue is still untested.
